Приложение ыполняет пинг всех серверов, указанных в JSON-файле, и возвращает статусы их доступности в формате JSON.

В первых строках приложения происходит импорт необходимых пакетов, таких как "encoding/json" для работы с JSON-данными, "github.com/labstack/echo/v4" для работы с веб-фреймворком Echo, "net" и "net/http" для работы с сетью и HTTP-протоколом, "os" для работы с операционной системой, "sync" для работы с синхронизацией и "time" для работы с временем.

Затем объявляются структуры Server, PingResult и PingResults, которые используются для хранения информации о серверах и результатах пинга.

Функция main начинается с чтения JSON-файла со списком серверов и декодирования его содержимого в переменную servers. Затем создается экземпляр веб-фреймворка Echo, который обрабатывает GET-запросы на пути "/ping".

При получении такого запроса происходит пинг всех серверов, указанных в переменной servers. Для этого используется группа горутин и объект PingResults для синхронизации доступа к результатам пинга. В цикле for происходит запуск горутин для каждого сервера, которые пингуются параллельно. Результаты пинга сохраняются в общем объекте PingResults.

После завершения всех пингов возвращаются результаты в формате JSON.

Функция ReadFile отвечает за чтение файла по указанному пути и возвращает его содержимое в виде среза байтов ([]byte). Она используется для чтения JSON-файла со списком серверов.
